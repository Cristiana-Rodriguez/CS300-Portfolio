Project Artifacts

Reflection

What was the problem you were solving in the projects for this course?

The primary problem I solved in this course was designing a program that allows academic advisors to efficiently manage and retrieve course information for the ABC University Computer Science program. The application needed to read course data from a file, organize it using an appropriate data structure, print the complete course list in alphanumeric order, and quickly search for information about individual courses and their prerequisites.

How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by comparing several data structures, including vectors, hash tables, and binary search trees. I analyzed the runtime and memory requirements of each structure before deciding which one best fit the project requirements. Understanding data structures is important because different structures provide different advantages depending on the operations being performed. Choosing the correct data structure can significantly improve the efficiency, organization, and scalability of a program.

How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was correctly loading and validating the course data from the CSV file while ensuring prerequisites were stored properly. I also encountered issues with file paths and debugging my binary search tree implementation. I overcame these challenges by testing the program incrementally, verifying each function before moving on, and carefully debugging compiler errors and program output until everything worked as expected.

How has your work on this project expanded your approach to designing software and developing programs?

This project helped me understand that software design begins long before writing code. Creating pseudocode, comparing data structures, and analyzing algorithm efficiency made me think more carefully about design decisions before implementation. I now recognize the importance of selecting appropriate algorithms and data structures early in the development process to produce software that is both efficient and maintainable.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This course strengthened my programming practices by encouraging modular design and clear organization. I separated functionality into individual functions, used meaningful variable names, included comments, and designed reusable code that can be modified more easily in the future. These practices make programs easier to read, debug, maintain, and expand as project requirements change.
